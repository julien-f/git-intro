Why git?

Why version control?

  • Checkpoints: you can always go back!
  • Analysis: you can see what changed, and when!
  • Team Work: no more lost code!

Why version control? (bis)

  • Automated Bug finding (git bisect)
  • Multiple version (git branch)
  • Find who did what (git blame)
  • And much moreâ€¦

Why distributed?

Why distributed?

  • Local data: server failure resistant!
  • Local branching: be messy!
  • No central point: fork at will!

Why git?

  • High performance (even on big projects)
  • Flexible: everything can be done (even if hard)
  • Most used DVCS

Developped by Linus Torvalds for the Linux Kernel.

See more at Why git is better.

How?

With great power there must also come great responsability!

git is extremely powerful but can sometimes be hard to master.

Getting help


git
git help <command>

First-time setup


git config --global user.name "Julien Fontanet"
git config --global user.email "julien.fontanet@vates.fr"

git config --global color.ui auto
git config --global push.default upstream

Repository creation


git init [<directory>]

Adding files


git add <files>

git commit

Checking the current status


$ git status
# On branch master
nothing to commit (working directory clean)

$ touch test
$ git status
# On branch gh-pages
# Untracked files:
#   (use "git add <file>..." to include in what will becommitted)
#
#	test
nothing added to commit but untracked files present (use "git add" to track)

Ignoring some files

Add entries to the .gitignore file:

/test

$ git status
# On branch master
nothing to commit (working directory clean)
